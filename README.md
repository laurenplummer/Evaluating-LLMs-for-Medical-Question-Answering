        "- **Size**: 246,678 examples\n",
        "- **Format**: Instruction + Input → Output (doctor's response)\n",
        "- **Language**: English\n",
        "- **Domain**: Medical QA from various sources (healthcare websites, medical forums)\n",
        "\n",
        "### 🔀 Data Split\n",
        "- **Train**: 197,342 examples (80%)\n",
        "- **Validation**: 24,667 examples (10%)\n",
        "- **Test**: 24,669 examples (10%)\n",

        "### 📈 Key Statistics\n",
        "- **Median Input Length**: 81 words\n",
        "- **Median Output Length**: 69 words\n",
        "- **95th Percentile Input**: 283 words\n",
        "- **Unique Input Vocabulary**: ~52,000 tokens (sampled)\n",
        "- **Duplicate Outputs**: 8.61% (21,227 examples)\n",
        "\n",
        "## 🔍 Exploratory Data Analysis (EDA)\n",
        "\n",
        "The EDA notebook ([EDA_Medical_QA.ipynb](EDA_Medical_QA.ipynb)) performs comprehensive analysis including:\n",
        "\n",
        "- **Data Quality Checks**: Missing values, duplicates, empty strings\n",
        "- **Text Analysis**: Length distributions, word counts, tokenization\n",
        "- **Vocabulary Analysis**: Token frequency, coverage, stopwords\n",
        "- **Visualization**: Histograms, scatter plots, word clouds\n",
        "\n",
        "### ✔ Key Findings\n",
        "- ✅ Dataset is clean with no missing values\n",
        "- ✅ Text lengths are manageable (95% fit within 256 tokens)\n",
        "- ✅ Rich medical vocabulary with good coverage\n",
        "- ⚠️ Some outputs are very short or generic\n",
        "- ⚠️ 8.61% duplicate responses indicate potential quality issues\n",
        "\n",
        "## 🤖 Modeling Approach\n",
        "\n",
        "### 1️⃣ LSTM Baseline Model\n",
        "- **Architecture**: Bidirectional LSTM with embedding layer\n",
        "- **Task**: Sequence-to-sequence generation\n",
        "- **Loss Function**: Cross-entropy\n",
        "- **Evaluation**: Perplexity, ROUGE-1/2/L scores\n",
        "- **Training**: 3 epochs, batch size 32, Adam optimizer\n",
        "\n",
        "### 2️⃣ Transformer Model (DistilGPT-2)\n",
        "- **Architecture**: DistilGPT-2 (distilled GPT-2)\n",
        "- **Task**: Causal language modeling\n",
        "- **Fine-tuning**: Using Hugging Face Trainer\n",
        "- **Training**: 3 epochs, batch size 16, learning rate 5e-5\n",
        "- **Features**: Pre-trained on large corpus, attention mechanism\n",
        "\n",
        "## 📈 Results & Model Comparison\n",
        "\n",

        "| DistilGPT-2   | ~82M      | TBD      | TBD     | TBD    | TBD    | TBD    |\n",
        "\n",
        "### 📊 Performance Insights\n",
        "- **Transformer Superiority**: Lower perplexity and higher ROUGE scores\n",
        "- **Quality**: Transformer generates more fluent, contextually appropriate responses\n",
        "- **Efficiency**: LSTM trains faster but performs worse on complex medical queries\n",
        "\n",
        "## ⚠️ Error Analysis\n",
        "\n",
        "### Common Issues Identified:\n",

        "- **Length Correlation**: Poor performance on very long input sequences (>200 tokens)\n",
